# Tag Note: v0.1.6

**Date**: 2026-04-08

`v0.1.6` records the current `flx` internal-first facade migration. It is an integration tag note rather than a formal release note. This tag freezes the package-layout cleanup that moves most implementation details under `internal/...` while keeping the root package focused on a thin public facade.

## Changed

- Moved the main implementation into `internal/collections`, `internal/config`, `internal/control`, `internal/runtime`, `internal/state`, and `internal/streaming`.
- Reduced the root production surface to the facade-oriented files `control_api.go`, `runtime_api.go`, `stream_api.go`, `transform_api.go`, and `doc.go`.
- Grouped root tests by facade area through `control_api_test.go`, `runtime_api_test.go`, `stream_api_test.go`, `transform_api_test.go`, and shared support in `test_support_test.go`.
- Unified internal import aliases after the migration so the new package layout reads consistently.
- No intended public API behavior change is introduced by this tag; the focus is structure, visibility boundaries, and root-package hygiene.
