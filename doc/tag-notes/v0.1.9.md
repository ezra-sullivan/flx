# Tag Note: v0.1.9

**Date**: 2026-04-10

`v0.1.9` records the imgproc example family cleanup, the public runtime observability-hook refinements, and the CI lint integration that now guards the coordinator/runtime code paths more strictly.

## Changed

- Replaced the old `http_image_pipeline` example line with the split imgproc family:
  - `examples/imgproc_pipeline` for the default business pipeline path
  - `examples/imgproc_observe` for observe-only usage
  - `examples/imgproc_coordinator` for coordinator control-loop usage
  - `examples/imgproc_retry` for deterministic local retry scenarios
- Moved shared example business logic into `examples/internal/imgproc`, restored the main list/download path to real `picsum.photos`, and kept only upload plus retry transport local and controlled.
- Added stable `kind=` log fields plus separate snapshot / decision formatting so coordinator logs are easier to distinguish from ordinary item and retry logs.
- Refined the public runtime APIs so they no longer log on behalf of callers by default, promoted `control.ErrorStrategyContinue`, and added retry / timeout observer hooks through `WithOnRetry(...)` and `WithTimeoutLatePanicObserver(...)`.
- Hardened controller/coordinator behavior around forced shrink registration and serialized resource polling within one coordinator instance.
- Added pinned `golangci-lint` configuration and a dedicated CI lint job.
