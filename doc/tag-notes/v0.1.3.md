# Tag Note: v0.1.3

**Date**: 2026-03-27

`v0.1.3` 当前作为 tag note 记录，而不是正式 release note。
它对应一组短路终结语义修复、错误可见性修复和配套文档收口，用于保留这个集成点的范围与验证结果。

## Fixed

- `Head` 现在会在关闭自身输出前先 drain 上游。
  这避免了 `Head(n)` 在并发 worker 场景下让下游过早完成、从而看不到晚到的 fail-fast error。
- `First` / `AllMatch` / `AnyMatch` / `NoneMatch` 现在会同步 drain 上游并在 drain 后再决定是否 panic。
  这样非 `*Err` 短路终结操作优先保证 fail-fast 错误可见性，不再吞掉晚到的 worker error。
- `FirstErr` / `AllMatchErr` / `AnyMatchErr` / `NoneMatchErr` 现在是真正短路。
  命中结果后会立即返回，并在后台继续 drain 上游。
- `ring` 的写入索引现在会在满环后有界回绕，不再依赖无限递增再取模。
  这移除了理论上的长期整数溢出路径。

## Changed

- 短路终结操作的语义现在分层明确：
  - 非 `*Err` 短路终结操作优先保证 fail-fast 错误可见性，因此会同步 drain 上游。
  - `*Err` 短路终结操作优先保证低延迟返回，因此返回的是当前错误快照，而不是最终错误集合。
- `README.md`、`doc/guide.md`、`doc/architecture.md`、`doc/fx-to-flx-migration.md` 已同步更新，明确上述语义边界和选型建议。

## Tests

- Added regression coverage for:
  - late fail-fast visibility on non-`*Err` short-circuit terminals
  - `Head(1).CollectErr()` observing delayed worker errors
  - true short-circuit behavior for `FirstErr` / `AllMatchErr` / `AnyMatchErr` / `NoneMatchErr`
  - continued upstream draining after early return for all short-circuit terminals
  - bounded wraparound behavior in `ring`
