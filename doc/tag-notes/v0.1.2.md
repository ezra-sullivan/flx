# Tag Note: v0.1.2

**Date**: 2026-03-26

`v0.1.2` 当前作为 tag note 记录，而不是正式 release note。
它对应一组运行时正确性修复和窗口化 API 落地，用于保留这个集成点的范围与验证结果。

## Added

- Count-based tumbling window APIs:
  `DistinctByCount` and `GroupByCount`.
- Processing-time tumbling window APIs:
  `DistinctByWindow` and `GroupByWindow`.
- New `Group[K, T]` result type for windowed grouping so grouped output keeps its key explicitly.
- New validation errors:
  `ErrInvalidWindowCount` and `ErrInvalidWindowDuration`.
- Documentation for `DistinctBy` / `GroupBy` memory boundaries and the new windowed APIs in `README.md` and `doc/guide.md`.

## Fixed

- `MapContext` and `MapContextErr` now send mapped results through `SendContext`, so blocked downstream sends can observe `ctx.Done()`.
  This fixes a worker-slot leak / shutdown stall path when `WithForcedDynamicWorkers` shrinks under backpressure.
- `Reduce` now drains the remaining upstream input after the reducer returns.
  Early-return reducers no longer leave upstream producers or transforms blocked.
- `From` now records producer panics into stream state instead of only logging and closing the source.
  `CollectErr` / `DoneErr` now surface these failures, and non-`*Err` terminals preserve fail-fast behavior.
- `First`, `AllMatch`, `AnyMatch`, and `NoneMatch` now honor already-recorded fail-fast errors instead of returning successfully after a short-circuit path.

## Tests

- Added regression coverage for producer panic propagation, short-circuit fail-fast behavior, forced dynamic worker shrink under blocked sends, early-return `Reduce` drain behavior, and all new windowed distinct/group APIs.
